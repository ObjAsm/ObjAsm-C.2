ObjAsm manual setup:

AUTOMATED INSTALLATION (recommended):
============================================================================

Run "...\ObjAsm\Build\OA_INST.cmd" for an interactive guided installation.
It performs all steps below automatically, verifies prerequisites, downloads
UASM, sets environment variables, and asks before optional compilation steps.


MANUAL INSTALLATION:
============================================================================

If you prefer to install manually, follow these instructions:


Prerequisites (skip to point 4 if you already have the proper tool chain)
============================================================================

1. Get Visual Studio Community (2017 or later) from
   https://www.visualstudio.com/ and install it.
   This provides the x64 debugger, linker, lib.exe, and the latest MSVC tool chain.

2. Get the Windows SDK from
   https://developer.microsoft.com/en-us/windows/downloads/windows-sdk/
   and install it. This provides the resource compiler (rc.exe), midl.exe,
   and other build utilities.

3. (Optional) Get the EFI Toolkit II from
   https://github.com/tianocore/tianocore.github.io/wiki/EDK-II
   and follow the instructions from
   http://www.terraspace.co.uk/UASM%20UEFI%20Guide.pdf


ObjAsm Installation
============================================================================

4. Get the ObjAsm package from http://objasm.x10host.com and install it.
   Depending on the installation drive and subdirectory, the OBJASM_PATH
   environment variable must be adjusted (see step 6).

5. Get the most recent UASM64.exe and UASM32.exe:

   AUTOMATIC (recommended):
     Run "...\ObjAsm\Build\Tools\Assembler\Download_UASM.cmd"
     This script downloads and installs both UASM32.exe and UASM64.exe
     automatically from terraspace.co.uk using PowerShell.

   MANUAL:
     Download from http://www.terraspace.co.uk/uasm.html
     and place UASM32.exe and UASM64.exe in the
     "...\ObjAsm\Build\Tools\Assembler" directory.

6. Create or update the following environment variables:

     (REQUIRED) OBJASM_PATH = C:\ObjAsm
                Set this to your actual ObjAsm installation root directory.

     (OPTIONAL) OBJASM_VS_PATH = C:\Program Files\Microsoft Visual Studio\2022\Community
                Pins a specific Visual Studio installation. Set automatically
                by OA_INST.cmd. Without this variable, OA_SET.cmd auto-detects
                the latest VS installation via vswhere.exe.

     (OPTIONAL) EFI_TOOLKIT_PATH = D:\EFI_Toolkit_2.0
                Only needed for UEFI development.

   Note: The MSVS_PATH and WINKIT_PATH variables are NO LONGER required.
   The build script OA_SET.cmd now auto-detects Visual Studio and the
   Windows SDK at runtime (see step 7).

   Reboot or log off/on for the environment variables to take effect.

7. OA_SET.cmd - Automatic Tool Detection (no manual path editing needed)

   The file "...\ObjAsm\Build\OA_SET.cmd" now dynamically detects:
     - Visual Studio installation path:
       * Uses OBJASM_VS_PATH if set and valid (pinned by OA_INST.cmd)
       * Falls back to vswhere.exe auto-detection (latest installation)
     - The latest MSVC toolset version (lib.exe, devenv.exe)
     - Windows SDK installation path and version (via registry)

   You do NOT need to manually edit paths in OA_SET.cmd anymore.
   It will automatically find the correct tools regardless of your VS edition
   (Community, Professional, Enterprise) or version (2019, 2022, etc.).

   If detection fails, OA_SET.cmd prints a clear error message indicating
   what is missing.

8. Register DebugCenter.exe by running it once:
   "...\ObjAsm\Projects\X\DebugCenter\DebugCenter.exe"

9. (Optional) Compile the ObjMem library:
   "...\ObjAsm\Code\ObjMem\MakeObjMem3264.cmd"

10. (Optional) Compile the objects:
    "...\ObjAsm\Code\Objects\Build\MakeObjects3264AW.cmd"

11. (Optional) Compile the Examples and Projects:
    "...\ObjAsm\Examples\MakeAll.cmd"
    "...\ObjAsm\Projects\MakeAll.cmd"


Troubleshooting
============================================================================

- If OA_SET.cmd reports "vswhere.exe not found":
  Ensure Visual Studio 2017 or later is installed. The VS Installer places
  vswhere.exe at: %ProgramFiles(x86)%\Microsoft Visual Studio\Installer\

- If OA_SET.cmd reports "Windows SDK not found":
  Install the Windows 10/11 SDK from the link in step 2, or via the
  Visual Studio Installer (Workloads > Desktop development with C++).

- If OA_SET.cmd reports "OBJASM_PATH not set":
  Set the environment variable as described in step 6.
