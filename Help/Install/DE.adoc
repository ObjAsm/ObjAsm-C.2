=== LNK1102 "Nicht gen³gend Speicher" mit VS2022/VS2026 Linker

Beim Linken von Debug-Builds mit `link.exe` aus Visual Studio 2022 oder neuer kann folgender Fehler auftreten:

----

**Ursache:** UASMs altes `-Zi` Debug-Format erzeugt CodeView-Records, die neuere MS-Linker nicht korrekt verarbeiten k÷nnen. Dies ist ein bekanntes UASM-Problem (https://github.com/Terraspace/UASM/issues/194[GitHub #194]).

**Alternative:** Den mit ObjAsm mitgelieferten Linker unter `Build\Tools\Linker\link.exe` verwenden, der sowohl mit `-Zi` als auch `-Zi8` Output kompatibel ist.
